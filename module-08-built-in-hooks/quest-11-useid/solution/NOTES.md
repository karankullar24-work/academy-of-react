# Quest 11 Solution: Accessible Spell Form with useId

## Overview

A spell registration form where each label/input pair has a unique, stable ID generated by `useId`. This ensures proper accessibility and works safely with server-side rendering.

## Key Concepts

### 1. Generating Unique IDs

```javascript
const nameId = useId();
const elementId = useId();
const powerId = useId();
```

Each call to `useId()` returns a different unique string (like `:r1:`, `:r2:`). The IDs are stable across re-renders and unique across component instances.

### 2. Label–Input Association

```jsx
<label htmlFor={nameId}>Spell Name:</label>
<input id={nameId} type="text" />
```

Screen readers use the `htmlFor`/`id` connection to announce labels when inputs are focused. Hardcoded IDs break when a component is rendered twice on the same page.

### 3. SSR Safety

Unlike `Math.random()` or counters, `useId` generates deterministic IDs that match between server and client hydration — no mismatch warnings.

### 4. Multiple Instances

If you render two copies of the same form component, each gets its own set of IDs. No collisions, no manual prefixing needed.

## Why Not Just Hardcode IDs?

| Approach            | Multiple instances? | SSR-safe? |
| ------------------- | ------------------- | --------- |
| Hardcoded `id`      | Collisions          | Yes       |
| `Math.random()`     | Unique              | No        |
| Counter / `useRef`  | Unique              | No        |
| `useId()`           | Unique              | Yes       |

## Testing

1. Click a label — the associated input should focus
2. Inspect the DOM — IDs are unique strings (`:r0:`, `:r1:`, etc.)
3. Render the form twice — no duplicate IDs
